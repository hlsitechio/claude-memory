## Memory System v2 — state.md (YOUR EXTERNAL BRAIN)

### The Architecture

**Anthropic's compact is a black box — we can't control it.**
But we CAN make it irrelevant. `state.md` lives on disk. It survives compact untouched.
After compact, you READ it back. Full recovery.

```
Working → Update state.md → Compact fires → state.md still on disk → Read it → Resume
```

### state.md — The One File That Matters

Location: `{session_path}/state.md`

```markdown
# Session State
> Last updated: HH:MM

## Goal
What we're doing and why.

## Progress
- [x] Completed task (with key result)
- [ ] Current task
- [ ] Next task

## Findings
### Finding 1: Title
- Evidence: ...
- Status: confirmed/potential
- Impact: ...
```

### How to Maintain state.md

1. **Session starts** → state.md has a template. Update **Goal** when user gives direction.
2. **As you work** → Use **Edit tool** to update Progress (check items, add new ones) and Findings (add discoveries).
3. **Every 3-5 tool calls** → Quick state.md update. Keep it current.
4. **Before compact** → The hook snapshots state.md to .mci automatically.
5. **After compact** → Read state.md → you have everything → resume from Progress checklist.

### Display Markers (output formatting)

| Marker | Meaning |
|--------|---------|
| `[+]` | Success/found |
| `[-]` | Failed/not found |
| `[!]` | Critical find → Edit **Findings** section |
| `[>]` | Next step → Edit **Progress** section |
| `[*]` | Context/goal shift → Edit **Goal** section |
| `[i]` | Info/observation → Append to memory.md |

### Session Files

```
{session_path}/
├── state.md      ← YOUR EXTERNAL BRAIN (living state, Edit to update)
├── memory.md     ← Session log (append-only, [i] entries)
└── memory.mci    ← Safety net snapshot (auto-generated by hooks)
```

### Post-Compact Recovery (CRITICAL)

When you detect post-compact (hook tells you, or context feels empty):

1. **Read `{session_path}/state.md`** — your full state is there
2. **Check the Progress section** — see what's done, what's next
3. **Resume from the first unchecked item** in Progress
4. **DO NOT ask the user what you were doing** — state.md tells you

### Drawer Model
SessionStart loads ONLY: identity (SOUL+USER) + latest .mci (~800 tokens total)
Everything else is a **drawer** you open on demand:
- `state.md` → Read after compact for full recovery (YOUR BRAIN)
- `MEMORY.md` → Read when you need long-term knowledge
- `memory/*.md` → Read when you need daily notes
- Memory Engine MCP tools → Search when you need past conversations
- `compact-*.md` → Read when you need raw conversation backup

### Permanent Knowledge Rules
- SOUL.md and USER.md define who you are. Read them. Embody them.
- When the user expresses preferences, update USER.md
- If the user says something PERMANENT (not session-specific), update MEMORY.md directly
- state.md is SESSION-specific. MEMORY.md is PERMANENT.
